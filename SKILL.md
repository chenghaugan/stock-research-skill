# 个股深度研究报告 Skill

全自动生成专业投资深度研报，覆盖 A 股 / 港股 / 美股。

## When to Activate

用户要求：
- 写一篇XX公司的深度研究报告/个股研报/估值分析
- 分析XX公司的投资价值/基本面/竞争力
- 生成XX股票的深度报告

## Core Parameters

用户必须提供：
- **股票代码**（必填）：如 `000001.SZ`、`0700.HK`、`AAPL`
- **股票名称**（必填）：如"平安银行"
- **输出路径**（可选）：默认 `./research-output/<TICKER>/`

## Market Identification

根据代码格式自动识别市场：

| 市场 | 代码格式 | 数据 Skill | 识别规则 |
|------|---------|-----------|---------|
| A 股 | 6位数字 | `a-stock-data` | 以 6/9 开头→上海, 以 0/3 开头→深圳, 以 8 开头→北交所 |
| 港股 | 5位数字.HK | `global-stock-data` | 包含 `.HK` 后缀或 4-5位数字+HK |
| 美股 | 字母代码 | `global-stock-data` | 纯英文字母代码（1-5个字符） |

## Workflow

### Step 1: Initialize

1. 解析用户输入的股票代码和名称
2. 识别市场类型
3. 确定输出根目录：`./research-output/<TICKER>/`
4. 创建目录结构：
   ```
   ./research-output/<TICKER>/
   ├── data/        ← 原始采集数据
   └── output/      ← 最终报告
   ```
5. 加载对应市场的 data-checklist reference 文件：
   - A股 → `references/a-share-data-checklist.md`
   - 港股 → `references/hk-stock-data-checklist.md`
   - 美股 → `references/us-stock-data-checklist.md`

### Step 2: Data Collection (4-Lane Parallel)

根据加载的 data-checklist reference 文件，并行启动 4 条采集线路。

**采集优先级原则**：
- 日期越近的信息权重越高（见 Step 3 校验规则）
- 竞对公司数据必须同步采集
- 所有采集结果保存到 `data/` 子目录，文件名包含采集类型和时间戳

**Lane 1 — 行情 + 财务数据**

执行 data-checklist 中"行情与财务"部分的所有端点。

**Lane 2 — 研报 + 一致预期**

执行 data-checklist 中"研报与预期"部分的所有端点。使用 Tavily 搜索补充：
- `"[公司名称] 券商研报 目标价 最近3个月"`
- `"[公司名称] 业绩交流会 纪要"`

**Lane 3 — 新闻 + 公告**

执行 data-checklist 中"新闻与公告"部分的所有端点。使用 Tavily + web-access 补充搜索最新新闻和管理层动态。

**Lane 4 — 行业 + 竞争**

使用 Tavily 搜索以下方向（采集结果保存到 data/industry_<timestamp>.md）：
- `"[行业名称] 市场规模 行业增速 2025 2026"`
- `"[行业名称] 竞争格局 市场份额"`
- `"[公司名称] 竞争对手"`
- `"[公司名称] SWOT分析"`

**竞对公司采集（⚠️ 强制步骤，不可跳过）**：

从 Lane 4 行业搜索中确定 3-5 家可比公司后，执行 data-checklist 中
"四、行业与竞争"下的竞对公司采集端点。**每家竞对至少采集以下字段**：

| 字段 | A股端点 | 港美股端点 |
|------|---------|-----------|
| PE(TTM)、PB、市值 | 腾讯财经 API | 新浪行情 / Yahoo |
| 营收、净利、ROE | mootdx finance | Yahoo quoteSummary |
| 一致预期 EPS | 同花顺 THS | 东财datacenter / Tavily |

所有竞对数据保存到 `data/competitors_<timestamp>.md`，文件头部标注每家竞对的
采集时间戳和端点来源。

> ⚠️ **数据完整性门禁**：如果某家竞对的某个字段确实无法获取（如无机构覆盖一致预期），
> 必须标注"未获取"并说明原因，不得留空或以"—"占位。任一竞对的 PE/PB/市值/营收
> 四项全部缺失 → Lane 4 视为不完备，触发 Step 3 校验失败。

### Step 3: Data Freshness Validation

采集完成后执行校验。**不通过则重新采集，最多重试 2 次。**

| 检查项 | 标准 | 不通过处理 |
|--------|------|-----------|
| 股价/市值 | 当日或最近交易日 | 重新获取行情 |
| 竞对数据 | ≥3家竞对的PE/PB/市值/营收四项均不为空 | 重新采集缺失竞对 |
| 财务数据 | 最新报告期，距截止日 ≤6 个月 | 告警标注"财务数据已过期XX天" |
| 一致预期 | 预测期覆盖当前财年或下一财年 | 重新获取 |
| 研报销效 | 优先采用 ≤6 个月内的研报 | 超过6个月的标记为"仅历史参考" |

校验通过后，按信息时间加权标记每条数据：
- **≤7 天（3x 权重）**：报告核心依据，必须优先体现
- **≤30 天（2x 权重）**：重要参考，优先引用
- **≤6 个月（1x 权重）**：常规参考
- **>6 个月（0.5x 权重）**：仅历史背景，不作投资判断依据

### Step 4: AI Analysis

基于采集数据，依次完成以下分析。加载 `references/report-structure.md` 获取各章节详细要求。

> ⚠️ **分析前必读**：`references/writing-style.md` 中的「深度强制规则（三、深度强制规则）」— 每条关键判断必须走完"数据→因果→含义"三层。

**4.0 核心判断提炼（预写作，内部笔记，不输出到报告）**

在开始正式分析前，先列出全文的核心判断。这些判断是后续所有章节的"锚"，确保全文围绕同一逻辑主线展开：

```
内部判断笔记（不输出到报告）：

1. 一句话判断：这家公司值不值得看？为什么？
2. 最关键的3个判断（每项≤30字）：
   - 关于行业的判断：
   - 关于公司的判断：
   - 关于估值的判断：
3. 市场的主流分歧在哪？我们凭什么不一样？
4. 乐观情景成立的条件链：XX → XX → XX
5. 悲观情景触发的条件链：XX崩塌 → XX恶化 → 估值压缩
```

**4.1 主营业务拆解**
- 根据最新财报数据，按产品/服务线拆解收入构成（引用具体数字和占比）
- 标注同比/环比变化
- 识别收入驱动力

**4.2 业务分类**
- **核心基本盘**：当前收入贡献主力（占比>60%或利润主要来源），特征：成熟、现金流稳定
- **未来潜在看点**：当前占比较小但增速快（增速>30%），特征：高成长、战略意义

**4.3 行业分析**
- 行业空间（TAM/SAM）、增速（历史+未来预测）、格局（集中度/主要玩家）、特性（周期/政策/壁垒）

**4.4 竞争力判断**
- 定性分析为主（技术、渠道、品牌、成本优势），定量对比竞对公司财务数据
- 引用竞对公司最新财报数据做横向对比表

**4.5 投资逻辑提炼**
- **核心逻辑**：详细描述最关键的看涨/看跌理由，每段有逻辑链
- **逆向思辨（硬性要求）**：
  - 解释"市场为什么错"——市场定价的逻辑是什么，我们凭什么不同
  - 设定证伪条件——什么情况下我们的判断是错的
  - 回应至少一个市场主流分歧观点
- **催化剂**：近期（6月内）+ 中期（1-2年），必须有具体时间节点
- **关键假设**：估值成立的 2-3 个核心前提，每个带有可验证的阈值

**4.6 估值讨论**
AI 根据公司特征自行判断，选择最合适的方法（可多方法交叉验证）：
- 稳定盈利 → **P/E** + 同行对比 + 历史估值区间
- 未盈利/高增长 → **P/S** + 增速调整
- 多元业务 → **SOTP** 分部估值
- 重资产 → **P/B**
- 所有方法可结合 **DCF** 作为锚定参考（自行设定增长率和折现率假设）

估值讨论要求（**禁止给出精确目标价**）：
- 乐观情景演绎：条件→估值区间→路径→概率（核心：市场定价逻辑怎么变）
- 悲观情景演绎：条件崩塌→估值杀到什么位置→概率
- 核心分歧：市场给当前估值的逻辑是什么，我们有什么不同
- 安全边际判断：风险收益比处于什么水平

**4.7 风险分级**
| 风险等级 | 定义 | 格式 |
|---------|------|------|
| 致命风险 | 投资逻辑崩塌级 | `<font color="red">**红色加粗**</font>` |
| 重大风险 | 显著影响估值 | **加粗** |
| 一般风险 | 常规经营风险 | 普通列表 |

每种风险必须附带：**监控阈值 + 触发后操作**

> ⚠️ **`<font>` 标签内 `<` 必须转义**：当 `<font color="red">...</font>` 内部包含"小于"符号（如 `营收增速<15%`），必须写成 `&lt;`（即 `营收增速&lt;15%`），否则会导致 Obsidian 渲染全部失效。详见 `references/writing-style.md` 7.1 节。

### Step 5: Report Writing

严格遵循 `references/writing-style.md` 的写作风格和 `references/report-structure.md` 的结构模版，生成最终报告。

**写作分两阶段：**

**Phase 1 — 主线写作**（核心章节，按叙事主线写）：
1. **投资研判框**（报告开头）：当前价位/乐观逻辑/悲观逻辑/核心分歧/关键跟踪指标
2. **公司简要介绍**
3. **主营业务分析**（基于最新财报数据）
4. **业务分类**（核心基本盘 + 潜在看点）
5. **行业深度分析**
6. **竞争力分析**（含竞对对比表）
7. **投资逻辑**（核心逻辑 + 逆向思辨 + 催化剂 + 关键假设）
8. **估值讨论**（乐观/悲观情景演绎 + 核心分歧 + 安全边际）

章节之间必须有过渡衔接，禁止孤立堆砌。每章牢记其在全文主线中的位置（行业逻辑 → 公司选择 → 估值锚定）。

**Phase 2 — 信息补充**（辅助章节，简洁即可）：
9. **风险关注**（分级 + 退出条件）
10. **重点跟踪**（指标 + 事件 + 需回答的问题）

**输出文件**：`output/<TICKER>_深度研究报告_<YYYYMMDD>.md`

**写作硬性约束**（来自 writing-style.md）：
- 禁止 AI 常用句式：值得注意的是、综上所述、我们可以看到、首先其次最后
- 有明确观点，不做和事佬
- 所有判断必须有数据支撑
- 核心观点加粗（全文加粗率 5-10%）
- 致命风险必须红色标注
- 每条关键判断走完"数据→因果→含义"三层
- 数据嵌入叙述中，后紧跟分析含义，不孤立列出
- 段落文字占每个章节 ≥ 80%，表格仅限竞对对比和风险分级
- 章节间用自然过渡语串联，章末有过渡句、章首有承接句
- 全文一条主线：行业逻辑 → 公司选择 → 估值锚定
- 投资逻辑必须包含逆向思辨（市场为什么错 + 证伪条件 + 回应分歧）
- 禁止给出精确目标价，用估值区间和情景演绎代替
- 避免"I think"、"I believe"等主观表述，用客观叙述代替

- [ ] 乐观/悲观情景演绎完整，逻辑链条清晰
- [ ] 安全边际评估到位
- [ ] 风险分级明确，致命风险红色标注
- [ ] 退出条件预设清晰
